Solution: White as a Sheet
Answer: WASTING

Written by Level 51

The puzzle page is completely blank, except for a link that sends solvers to a... also completely blank Google Sheets spreadsheet, or so it seems. This is a minimeta-style puzzle comprising seven minipuzzles; each is a separate tab in the linked spreadsheet.

Tab 1: Keyboards

Each of the slightly offset square patterns on the left represents a different keyboard layout, and we can observe that columns X, Y, and Z each contain four references to the layouts on the left, one per layout per column. Overlaying a typical QWERTY keyboard layout onto the topmost pattern indeed produces the letters X, Y, and Z in the respective columns, suggesting that those references always refer to X, Y, and Z in the corresponding layouts. The next three layouts similarly correspond to Dvorak, Workman, and Colemak respectively; overlaying all four layouts correctly produces the answer BECHAMEL from the references in column AA.

Tab 2: Pascal

This sheet simulates Pascal's triangle; each cell is the concatenation of the cell to its left and above it, and thus represents some number of copies of cell A1. Six cells have different formulas, instead giving a series of question marks with two in brackets, representing directly making some number of copies of A1. Finding this number for each of the six cells, we notice that it has the same number of digits as there are question marks in the formula. Extracting the two digits that correspond to the question marks in brackets, and converting via A1Z26, gives the subpuzzle answer RAPIDS.

Tab 3: Star Battle

This sheet comprises an upper and lower half; some examination of the formulas in the lower half indicate that they will produce a letter when something is filled into the corresponding square in the upper half, so we can consider it extraction and leave it alone. We might attempt filling things into the upper half; we notice that cells turn green when they're filled with exactly one character (besides the hyphen "-"). Some further experimentation reveals that some cells sometimes turn red. It turns out there are a few conditions for this:

  • There are more than two cells filled in a row or a column.
  • Two filled cells are adjacent (including diagonal adjacency).
  • Sometimes, three cells might turn red at the same time even though they're not in the same row or column.

Some thought about logic puzzles, or perhaps viewing the conditional formatting programmed into this sheet, reveals that the upper half of the sheet is simulating a Star Battle logic puzzle, where regional restrictions cause the strange behavior of three cells sometimes turning red. We can either derive the regions from viewing the conditional formatting or from experimenting with the board to find out when cells turn red. Either way, solving the logic puzzle and extracting the corresponding letters from the lower half of the sheet spells out SUBANSWER'S OVAL OFFICE, so the answer to this subpuzzle is OVAL OFFICE.

Tab 4: Pokemon

There are 151 rows in this tab, and the cells in column A are by and large grouped into groups of 1, 2 or 3 via border coloring. However, there's a noticeable instance where cell A133 looks "paired" with cell A134, A135, and A136. Some thought about what 151 could imply indicates that this represents the evolutionary lines of the first generation of Pokémon, where Pokémon 133, Eevee, evolves into Vaporeon, Jolteon, or Flareon. Cell D1 highlights eight cells from column A; looking up the respective Pokémon reveals that they are, in order, Flareon, Arbok, Scyther, Tentacool, Farfetch'd, Omastar, Onix, and Dewgong. Their initials spell out this subpuzzle's answer, FAST FOOD.

Tab 5: Crossword

The most suspicious thing about this tab is the strange column widths from column L onwards. Each of them seems to be a multiple of 10; dividing by 10 gets us numbers from 1 to 19, which we can convert via A1Z26 to read as NAMED RANGES. There are fourteen named ranges in this spreadsheet, and all of them appear on the left-hand side of this tab.

There are two different types of named ranges here: one type covers seven cells and appears either vertically or horizontally. There are eight of these, which form a crossword grid in the region from C2 to I8. The other type covers two cells and appear only horizontally, in one column; there are six of these from A1:B1 to A6:B6.

Tab 6: Formula

This is a very small sheet, with a blank cell A1 and a very long formula in cell B1. Solvers must deconstruct the formula in cell B1 to figure out exactly what it does. Here's a breakdown of what it does.

IFERROR
This is wrapped around the whole formula to suppress any errors from being shown.

IF
This starts a conditional statement, containing three parameters: a predicate, a value to output if the predicate is true, and a value to output if the predicate is false. Skipping to the end, we can see that both values are empty strings, but we could modify them to output various things for checking purposes later.

AND
The predicate begins with an AND statement, implying that it's only true if all conditional statements inside the AND are true. There are exactly three of them, separated by commas. Let's look at each one:

Condition 1: COLUMN(INDIRECT(LEFT(A1,3)&1))>12870

COLUMN returns the column number of a cell. For example, COLUMN(F1) returns 6, since column F is the sixth column. INDIRECT creates a reference to a cell from a string; this string is LEFT(A1, 3), which is the first three letters of the input word, plus "1". Therefore, the column represented by the first three letters of the input word must be later than the 12870th column. Some calculation shows that the 12870th column is RZZ, which implies that the first letter of the input word must be S or later.

Condition 2: CONCATENATE(ARRAYFORMULA(COUNTIF(MID(A1,SEQUENCE(LEN(A1)),1),MID(A1,SEQUENCE(LEN(A1)),1))))="113321123"

This is the hardest part of this subpuzzle, but it also gives essential information for solving it. Some ways to approach this are to extract this portion of the formula, applying it to various words of our choice, and seeing what it outputs. Here, we'll break down the formula into individual portions. This time, it mostly makes more sense to go from the inside out, so let's start there. One thing to note is the ARRAYFORMULA, which allows functions to act on arrays of cells instead of single cells.

SEQUENCE(LEN(A1))
SEQUENCE generates an array from 1 to its parameter P; here, P is LEN(A1), or the length of string A1. For example, if A1 is the word "TEST", this is just the sequence [1, 2, 3, 4], represented as an array.

MID(A1,SEQUENCE(LEN(A1)),1)
MID has a string parameter and two numerical parameters N and M; it returns the M characters starting at position N in the string. Since MID is inside an ARRAYFORMULA, it can accept a range of values of N. Here, A1 is the input string, and M is simply 1, but N is the sequence derived in the previous step. This therefore returns an array comprising each of the letters in A1, but as an array of characters instead of a single string.

COUNTIF
COUNTIF usually takes an array to query and a single query, and returns the number of items in the array that satisfy the query. However, since it's inside an ARRAYFORMULA, it can instead accept an array of queries. For each query, we're checking how many items in the array version of our input string match each of its letters. Consider the example array [T, E, S, T]; the first query, T, would return 2 since there are two Ts in this array, the second and third queries, E and S, would return 1 each, and the last query, T again, would also return 2.

ARRAYFORMULA
This is merely the modifier that has allowed our previous couple of functions to act on arrays rather than single parameters.

CONCATENATE
This implies we're receiving an array of data. The function simply concatenates all the data it receives into one long string.

With all this in mind, we can determine that we're looking for a string which outputs 113321123 from this formula. That means it's a nine letter word, its first, second, sixth, and seventh letters must be unique in the word, its fifth and eighth letters must match, and its third, fourth, and ninth letters must also match.

Condition 3: CODE(LEFT(A1))=CODE(RIGHT(A1))+1

CODE returns the ASCII codepoint represented by a character. This condition is equivalent to checking that LEFT(A1), which is A1's first letter, is exactly one ahead of RIGHT(A1), which is A1's last letter, in the alphabet, without wrapping. For example, FREE and TENS satisfy this, but SENT and ABUZZ do not.

All conditions

In combination, these conditions require the word to have a first letter S or later, a letter pattern of ABCCDEFDC, and a last letter which is one behind its first letter in the alphabet. Using a cryptogram solver like quipqiup with a query like ABCCDEFDC shows that the only common English word that satisfies these conditions is SURRENDER.

Tab 7: Columns

This sheet features columns in three different widths, encoding Morse code with the narrowest columns representing dots, the medium-width columns representing dashes, and the widest columns representing pauses between letters. We can decipher this to COLFOURTWONINETWO; column 4292 in a Google Sheets spreadsheet is FIB, which is this subpuzzle's answer.

Putting it all together

We have seven subpuzzle answers. Each of them relates to a phrase of the form "white (something)", as shown in the table below. Additionally, we haven't used the tab names yet, which each are a certain number of a different whitespace character. Indexing into the various white things by the number of characters in the respective title, and sorting by decreasing length of whitespace character, we extract the final answer to this puzzle, WASTING. (Note that the link on the puzzle page comprises precisely these seven whitespace characters, arranged in decreasing length, as a further hint towards the ordering.)

Author’s Notes

I've enjoyed the aesthetic of puzzles that give solvers very little visible information, like Galactic Puzzle Hunt's This Puzzle Intentionally Left Blank, but I found that they're often very technical to solve, and don't parallelize well across multiple solvers. This puzzle is the result of wanting to create a more accessible version of a "blank" puzzle that we could put earlier on in our hunt and hopefully have more teams work on and solve.

The Star Battle was the first subpuzzle to be written, and was inspired by a friend's demonstration that it was possible to code an Akari interface purely in Google Sheets formulas. I decided to adapt the idea to a puzzle type without any givens in cells, and so created this Star Battle. The original intention was to break up the conditional formatting formulas so the only way to determine regions was via experimentation, but I didn't want that one subpuzzle to be much more difficult than the other subpuzzles. (Also, it looked like a lot of work.)
